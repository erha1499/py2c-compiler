# 编译原理课程设计报告

## 1. 课程设计题目

本课程设计实现了一个类 C 语言子集编译器，项目名称为 AnanasCC。该编译器使用 Python 编写，能够对 C 语言风格的源程序进行词法分析、语法分析、语义分析、三地址码生成、LLVM IR 生成，并进一步调用 clang 生成 x86 汇编和可执行文件。

项目主要面向编译原理课程设计，重点展示一个源程序从字符流到 Token 序列、语法树、语义标注、中间代码以及目标代码的完整处理过程。

## 2. 课程设计目的

本课程设计的主要目的如下：

1. 理解编译器的基本组成和处理流程。
2. 掌握词法分析器如何将源程序划分为 Token。
3. 掌握语法分析器如何根据文法规则构造语法树。
4. 理解语义分析中的类型检查、作用域管理和符号表维护。
5. 掌握中间代码生成的基本思想，尤其是表达式、条件语句和循环语句的三地址码生成。
6. 了解 LLVM IR 与目标汇编代码生成的基本过程。
7. 通过实际代码加深对编译原理课程内容的理解。

## 3. 开发环境

本项目的主要开发与运行环境如下：

- 操作系统：macOS
- 开发语言：Python
- 运行方式：命令行运行
- 主要依赖库：
  - `lark==1.2.2`：用于词法分析和 LALR(1) 语法分析
  - `llvmlite==0.44.0`：用于生成和验证 LLVM IR
  - `tabulate==0.9.0`：用于格式化输出 Token 表和分析表
- 外部工具：
  - clang：用于将 LLVM IR 或汇编进一步编译链接为可执行文件
  - matplotlib：用于绘制语法树图片

依赖安装命令如下：

```bash
pip install -r requirements.txt
```

项目可通过以下方式运行：

```bash
python3 main.py
```

也可以使用模块入口编译指定源文件：

```bash
python3 -m compiler your_file.c -e
```

其中 `-e` 表示编译完成后立即执行生成的程序。

## 4. 需求分析

本项目实现的是一个类 C 语言子集编译器。结合项目中的 `compiler/lexer/lexicon.lark`、`compiler/parser/syntax.lark` 和相关语义分析代码，编译器主要支持以下功能：

1. 词法识别
   - 关键字：`int`、`float`、`char`、`bool`、`void`、`struct`、`union`、`enum`、`if`、`else`、`for`、`while`、`break`、`continue`、`return` 等。
   - 标识符：普通变量名、类型名、枚举常量名。
   - 常量：整数、浮点数、字符常量、字符串常量、`true`、`false`、`nullptr`。
   - 运算符：算术运算、关系运算、逻辑运算、赋值运算、自增自减、成员访问等。
   - 分隔符：括号、花括号、方括号、逗号、分号等。

2. 语法结构
   - 变量声明与初始化。
   - 数组声明与初始化。
   - 函数声明与函数定义。
   - 结构体、联合体和枚举定义。
   - 表达式语句。
   - `if / else` 条件语句。
   - `while` 和 `for` 循环语句。
   - `break`、`continue`、`return` 跳转语句。

3. 语义分析
   - 基本类型和构造类型管理。
   - 嵌套作用域符号表管理。
   - 变量、函数、类型、常量的符号记录。
   - 未声明标识符、重复声明、类型不匹配等语义错误检查。
   - 数组大小的常量表达式检查。
   - `break`、`continue` 是否位于循环内的检查。

4. 中间代码与目标代码
   - 使用 `Annotator` 生成教学展示用三地址码。
   - 使用回填机制处理条件跳转和循环跳转。
   - 使用 `Generator` 生成 LLVM IR。
   - 使用 `Optimizer` 对 LLVM IR 进行优化。
   - 使用 clang 生成汇编和可执行文件。

项目目前仍存在一些不足。例如错误恢复机制较简单，目标三元组在 `Generator` 中写为 Windows MSVC 风格，在 macOS 上运行时依赖 clang 自动覆盖目标平台；部分复杂 C 语言特性虽然有语法或类型支持，但完整性和兼容性仍有限。

## 5. 系统总体设计

编译器整体流程由 `compiler/compiler.py` 中的 `Compiler` 类串联。主流程如下：

```text
源程序
  -> Lexer.lex()
  -> Token 序列
  -> Parser.parse()
  -> CST / AST
  -> Analyzer.analyze()
  -> 带类型和符号信息的 AST
  -> Annotator.annotate()
  -> 带三地址码和回填信息的 AST
  -> Generator.generate()
  -> LLVM IR
  -> Optimizer.optimize()
  -> 优化后的 LLVM IR
  -> clang 生成汇编
  -> clang 链接生成可执行文件
```

项目主要目录结构如下：

```text
compiler/
  lexer/       词法分析模块
  parser/      语法分析模块
  tree/        AST 节点与 CST 到 AST 的转换
  semantic/    语义分析、符号表、类型系统、三地址码注释器
  ir/          LLVM IR 生成与优化
  x86/         调用 clang 生成汇编和可执行文件
  compiler.py  编译器主控流程
```

其中，三地址码和 LLVM IR 是两条相对独立的中间表示：

- 三地址码由 `compiler/semantic/annotator.py` 生成，主要用于课程设计展示和理解控制流回填过程。
- LLVM IR 由 `compiler/ir/generator.py` 生成，用于实际编译、优化和目标代码生成。

## 6. 词法分析设计

词法分析器位于 `compiler/lexer/lexer.py`，词法规则定义在 `compiler/lexer/lexicon.lark` 中。项目使用 Lark 的基础词法引擎，根据正则表达式对源代码进行扫描。

词法分析入口函数为：

```python
tokens = self.lexer.lex(code)
```

`Lexer.lex()` 将源代码字符串转换为 Token 列表。每个 Token 包含类型、值、行号和列号信息，后续语法分析器据此进行分析。

### 6.1 关键字和标识符

词法规则中将关键字单独定义，例如：

```text
INT     : "int"
FLOAT   : "float"
IF      : "if"
ELSE    : "else"
FOR     : "for"
WHILE   : "while"
RETURN  : "return"
```

标识符分为几类：

```text
TYPE  : /(?=[A-Z0-9]*[a-z])[A-Z][a-z0-9]*(?:[A-Z][a-z0-9]*)*/
IDENT : /[a-z][a-z0-9]*(?:_[a-z0-9]+)*/
IMM   : /[A-Z][A-Z0-9]*(?:_[A-Z0-9]+)*/
```

其中 `TYPE` 常用于结构体、枚举等类型名，`IDENT` 用于普通变量名和函数名，`IMM` 用于枚举常量等大写标识符。

### 6.2 常量、运算符和界符

项目支持整数、浮点数、字符和字符串常量：

```text
INTEGER
DECIMAL
CHARACTER
STRING
```

运算符包括：

```text
+  -  *  /  %
>  <  >= <= == !=
&& || !
= += -= *= /= %=
++ --
. ->
```

界符包括：

```text
( ) [ ] { } , ;
```

### 6.3 空白、换行和注释

词法规则中使用 `%ignore` 忽略空白符和注释：

```text
WS      : /[ \t\f\r\n]/+
COMMENT : /\/\/[^\n]*/ | /\/\*(.|\n)*?\*\//

%ignore WS
%ignore COMMENT
```

因此，空格、换行、单行注释和多行注释不会进入 Token 序列。

### 6.4 词法错误处理

当源代码中出现无法识别的字符时，Lark 会抛出 `UnexpectedCharacters`，项目将其转换为自定义的 `LexicalError`：

```python
raise LexicalError('无法识别的字符', e.line, e.column)
```

错误信息包含错误类型、行号和列号，便于定位问题。

## 7. 语法分析设计

语法分析器位于 `compiler/parser/parser.py`，文法规则位于 `compiler/parser/syntax.lark`。项目使用 Lark 的 `parser='lalr'`，即 LALR(1) 自底向上语法分析方式。

语法分析过程分为两步：

1. Token 序列根据 `syntax.lark` 生成 CST。
2. 使用 `compiler/tree/transformer.py` 中的 `ASTTransformer` 将 CST 转换为更适合后续处理的 AST。

核心入口如下：

```python
ip = self.parser.parse_interactive()
for token in tokens:
    ip.feed_token(token)
self.cst = ip.feed_eof(tokens[-1])
self.ast = ASTTransformer().transform(self.cst)
```

### 7.1 程序和声明规则

程序由多个编译单元组成：

```text
program : unit*
unit    : definition | declaration
```

定义和声明包括函数定义、结构体定义、枚举定义、函数声明、变量声明和数组声明。

### 7.2 语句规则

语句主要包括：

```text
statement : comp_stmt | expr_stmt | select_stmt | iter_stmt | jump_stmt
```

其中：

- `comp_stmt` 表示复合语句块。
- `expr_stmt` 表示表达式语句。
- `select_stmt` 表示 `if / else`。
- `iter_stmt` 表示 `while` 和 `for`。
- `jump_stmt` 表示 `break`、`continue` 和 `return`。

### 7.3 表达式优先级

项目通过分层文法体现表达式优先级：

```text
assign_expr
lor_expr
land_expr
equal_expr
rel_expr
add_expr
mul_expr
unary_expr
postfix_expr
primary_expr
```

这种设计将赋值、逻辑或、逻辑与、相等比较、关系比较、加减、乘除取模、一元运算、后缀运算和基本表达式分层处理，避免了在代码中手工维护复杂的优先级关系。

### 7.4 条件和循环语句识别

`if / else` 的文法规则如下：

```text
select_stmt : IF LPAREN expression RPAREN statement (ELSE statement)?
```

`while` 和 `for` 的文法规则如下：

```text
iter_stmt : WHILE LPAREN expression RPAREN statement
          | FOR LPAREN (expr_stmt | declaration) expression? SEMICOLON expression? RPAREN statement
```

其中 `for` 支持初始化部分为表达式语句或声明语句，条件表达式和更新表达式可以省略。

## 8. 语义分析与符号表

语义分析器位于 `compiler/semantic/analyzer.py`，符号表定义在 `compiler/semantic/symbol.py`，类型系统定义在 `compiler/semantic/type.py`。

### 8.1 符号表结构

项目中的 `SymbolTable` 使用作用域栈管理符号：

```python
class SymbolTable:
    def __init__(self):
        self.stack = []
        self.enter_scope()
```

每个作用域是一个字典。进入新的复合语句、函数或结构时可以创建新作用域，退出时弹出作用域。查找符号时从内层作用域向外层作用域查找：

```python
def lookup(self, name):
    for scope in reversed(self.stack):
        if name in scope:
            return scope[name]
    return None
```

符号对象 `Symbol` 主要记录以下信息：

- `type`：符号类型，例如 `int`、`float`、数组类型、函数类型等。
- `name`：符号名称。
- `kind`：符号类别，例如类型、函数、变量、常量。
- `node`：对应 AST 节点。
- `defined`：是否已定义。

### 8.2 类型系统

项目实现了较完整的类型类层次，包括：

- `BasicType`：基本类型，如 `int`、`float`、`char`、`bool`、`void`。
- `PointerType`：指针类型。
- `ArrayType`：数组类型。
- `FunctionType`：函数类型。
- `CompoundType`：结构体或联合体类型。
- `EnumType`：枚举类型。

语义分析器在初始化时会把内置类型和标准库函数加入符号表：

```python
for name, type in [('void', VOID), ('int', INT), ('float', FLOAT),
                   ('char', CHAR), ('bool', BOOL)]:
    self.table.define(Symbol(type, name, SymbolKind.TYPE))

self.table.define(Symbol(FunctionType(INT, None), 'printf', SymbolKind.FUNC))
self.table.define(Symbol(FunctionType(INT, None), 'scanf', SymbolKind.FUNC))
```

### 8.3 语义检查内容

项目语义分析主要完成以下工作：

1. 检查类型名是否已经声明。
2. 检查变量、函数、类型是否重复定义。
3. 检查变量使用前是否已经声明。
4. 检查赋值表达式左右类型是否兼容。
5. 检查数组大小是否为 `int` 类型常量表达式。
6. 检查初始化列表长度是否超过数组或结构体成员数量。
7. 检查 `return` 表达式类型是否与函数返回类型匹配。
8. 检查 `break` 和 `continue` 是否位于循环语句内部。

语义分析完成后，AST 节点会被标注 `ctype`、`symbol`、`index` 和 `attrs` 等信息，供后续三地址码生成和 LLVM IR 生成使用。

## 9. 中间代码生成

本项目中存在两类中间表示：

1. 教学展示用三地址码，由 `compiler/semantic/annotator.py` 中的 `Annotator` 生成。
2. 实际编译用 LLVM IR，由 `compiler/ir/generator.py` 中的 `Generator` 生成。

课程设计报告中重点说明三地址码生成，同时简要说明 LLVM IR 的作用。

### 9.1 三地址码形式

项目生成的三地址码格式类似：

```text
t1 = a + b
if t1 goto L1
goto L2
L1:
x = t1
```

它与常见四元式 `(op, arg1, arg2, result)` 对应关系如下：

```text
t1 = a + b    对应    (+, a, b, t1)
x = t1        对应    (=, t1, _, x)
if t1 goto L1 对应    (if, t1, _, L1)
```

项目中没有单独定义四元式类，而是以字符串形式输出三地址码，并保存到 `temp/06 tac.txt`。

### 9.2 临时变量和标签

`Annotator` 使用计数器生成临时变量和标签：

```python
def _next_tmp(self):
    self.temp_cnt += 1
    return f't{self.temp_cnt}'

def _next_label(self):
    self.label_cnt += 1
    return f'L{self.label_cnt}'
```

例如表达式：

```c
sum = sum + sts[i].score;
```

会生成类似三地址码：

```text
t10 = sts[i]
t11 = t10.score
t12 = sum + t11
sum = t12
```

### 9.3 条件语句的跳转生成

对于 `if / else` 语句，项目通过条件跳转和标签组织控制流。例如：

```c
if (score >= 60.0) {
    count++;
} else {
    ...
}
```

会生成类似：

```text
t4 = t3 >= 60.0
if t4 goto L5
goto L6
L5:
...
goto L7
L6:
...
L7:
```

其中 `L5` 是 then 分支入口，`L6` 是 else 分支入口，`L7` 是两个分支结束后的汇合位置。

### 9.4 循环语句的跳转生成

对于 `while` 和 `for`，项目会生成循环条件标签、循环体标签、更新标签和循环结束标签。

以 `for` 为例：

```c
for (; i < 4; i++) {
    ...
}
```

生成的三地址码中可以看到：

```text
L1:
t1 = i < 4
if t1 goto L2
goto L4
L2:
...
L3:
t13 = i
i = i + 1
goto L1
L4:
```

其中：

- `L1` 为条件判断位置。
- `L2` 为循环体入口。
- `L3` 为 `for` 的 update 部分。
- `L4` 为循环结束位置。

`for` 的 update 部分需要放在循环体之后，是因为 C 语言中 `for(init; condition; update) body` 的执行顺序为：

```text
init -> condition -> body -> update -> condition -> ...
```

如果把 update 放在循环体之前，就会改变程序语义，使第一次循环体执行前就更新循环变量。

### 9.5 LLVM IR 生成

LLVM IR 生成器位于 `compiler/ir/generator.py`。它将语义分析后的 AST 映射到 LLVM IR，包括：

- `int` 映射为 `i32`。
- `float` 映射为 `float`。
- `char` 映射为 `i8`。
- `bool` 映射为 `i1`。
- 指针、数组、结构体映射为 LLVM 对应类型。

例如 `main.c` 中的结构体：

```c
struct Student {
    char* name;
    int num;
    int age;
    float score;
};
```

生成的 LLVM IR 中出现：

```llvm
%"Student" = type {i8*, i32, i32, float}
```

LLVM IR 最终会被保存为 `temp/04 org_ir.txt`，优化后的 IR 保存为 `temp/04 opt_ir.txt`。

## 10. 错误处理

项目定义了统一的编译错误基类 `CompileError`，位于 `compiler/error/error.py`：

```python
class CompileError(Exception):
    def __str__(self):
        return f'{self.__class__.__name__}({self.line}, {self.column}): {self.msg}'
```

其子类包括：

- `LexicalError`：词法错误。
- `SyntaxError`：语法错误。
- `SemanticError`：语义错误。

### 10.1 词法错误

当源程序中出现词法规则无法识别的字符时，`Lexer.lex()` 会抛出：

```python
LexicalError('无法识别的字符', line, column)
```

### 10.2 语法错误

当 Token 序列不符合 `syntax.lark` 中定义的语法规则时，`Parser.parse()` 会抛出：

```python
SyntaxError('无法识别的单词', line, column)
```

例如括号不匹配、表达式缺失、语句结构不完整等情况都可能触发语法错误。

### 10.3 语义错误

语义错误由 `Analyzer.raise_error()` 抛出，常见情况包括：

- 未声明的类型或变量。
- 重复声明。
- 类型不匹配。
- 数组大小不是整型常量表达式。
- 初始化项数量不合法。
- `break` 或 `continue` 出现在循环外。

项目目前主要采用“发现错误后抛出异常”的方式处理错误，没有实现复杂的错误恢复机制。因此出现错误后通常会终止当前编译流程或打印错误信息。

## 11. 测试用例与运行结果

### 11.1 测试用例一：变量声明与赋值

输入源程序片段：

```c
void main() {
    int i = 0;
    int count = 0;
    float sum = 0;
}
```

词法分析输出中可以看到 `INT`、`IDENT`、`ASSIGN`、`INTEGER`、`FLOAT`、`SEMICOLON` 等 Token。三地址码输出为：

```text
.func main:
        .001: i = 0
        .002: count = 0
        .003: sum = 0.0
.ret
```

该用例说明编译器能够识别基本类型声明和初始化语句，并能为初始化生成赋值形式的三地址码。

### 11.2 测试用例二：表达式计算

输入源程序片段：

```c
sum = sum + sts[i].score;
```

在 `main.c` 的编译结果中，三地址码为：

```text
.030: t10 = sts[i]
.031: t11 = t10.score
.032: t12 = sum + t11
.033: sum = t12
```

该结果说明项目会先生成数组访问和成员访问的临时变量，再对表达式结果进行赋值。

### 11.3 测试用例三：if / else 条件语句

输入源程序片段：

```c
if (sts[i].score >= 60.0) {
    count++;
    printf("%s success\n", sts[i].name);
} else {
    printf("%s fail\n", sts[i].name);
}
```

生成的三地址码片段如下：

```text
.009: t2 = sts[i]
.010: t3 = t2.score
.011: t4 = t3 >= 60.0
.012: if t4 goto L5
.013: goto L6
.014: L5:
.015: t5 = count
.016: count = count + 1
...
.022: goto L7
.023: L6:
...
.029: L7:
```

该结果体现了条件判断、then 分支、else 分支和分支汇合位置的跳转关系。

### 11.4 测试用例四：for 循环

输入源程序片段：

```c
for (; i < 4; i++) {
    ...
}
```

生成的三地址码片段如下：

```text
.004: L1:
.005: t1 = i < 4
.006: if t1 goto L2
.007: goto L4
.008: L2:
...
.034: L3:
.035: t13 = i

.037: goto L1

```

该结果说明编译器能够生成循环条件判断、循环体、更新语句和循环结束位置的控制流。

### 11.6 错误输入示例

若输入源程序中出现无法识别的字符，例如使用不在词法规则中的非法符号，词法分析阶段会报告：

```text
LexicalError(line, column): 无法识别的字符
```

若输入语句结构不符合文法规则，例如括号或分号缺失，语法分析阶段会报告：

```text
SyntaxError(line, column): 无法识别的单词
```

若使用未声明变量、重复声明变量或类型不匹配，语义分析阶段会报告：

```text
SemanticError(line, column): 错误说明
```

这些错误信息都包含行号和列号，能够帮助定位源程序中的问题。

## 12. 不足与改进

本项目已经实现了较完整的编译流程，但仍存在以下不足：

1. 错误恢复能力有限  
   当前错误处理主要是发现错误后抛出异常，尚未实现多错误收集和错误恢复。后续可以在词法、语法和语义阶段加入更友好的错误提示和恢复策略。

2. 目标平台配置不够灵活  
   `compiler/ir/generator.py` 中的 `module.triple` 写为 Windows MSVC 风格，在 macOS 上编译时会出现目标三元组被覆盖的警告。后续可以根据当前操作系统自动设置目标平台。

3. 中间代码表示仍偏展示用途  
   `Annotator` 生成的是字符串形式的三地址码，没有单独设计四元式数据结构。若后续需要做更多中间代码优化，可以将三地址码抽象为结构化对象。

4. 复杂 C 语言特性支持仍有限  
   项目已包含指针、数组、结构体、联合体、枚举、函数调用等支持，但与完整 C 编译器相比仍有差距，例如复杂声明、隐式类型转换、标准库支持、预处理器、完整错误恢复等功能仍可继续完善。

5. 编译流程的异常控制可以进一步改进  
   `Compiler.compile()` 中对异常和后续阶段的衔接还可以更加严格，确保语义分析失败时不会继续使用不完整 AST 进入 IR 生成阶段。

## 13. 课程设计总结

本课程设计完成了一个类 C 语言子集编译器 AnanasCC。项目以 Python 为主要开发语言，使用 Lark 完成词法分析和 LALR(1) 语法分析，使用自定义 AST、符号表和类型系统完成语义分析，并通过 `Annotator` 生成教学展示用三地址码，通过 `Generator` 生成 LLVM IR，最后调用 clang 生成汇编和可执行文件。

通过本项目，可以较清晰地看到编译器各阶段之间的关系：词法分析负责识别单词，语法分析负责组织程序结构，语义分析负责检查类型和作用域，中间代码生成负责把高级语法结构转换为更接近机器执行的形式。尤其是在 `if / else`、`while` 和 `for` 等控制流语句中，标签、条件跳转和回填机制是实现的重点。

本次课程设计加深了对词法分析、语法分析、符号表、类型检查、中间代码生成和目标代码生成的理解。后续可以继续完善错误恢复、平台适配、结构化中间代码、代码优化和更多 C 语言特性，使编译器更加稳定和完整。
